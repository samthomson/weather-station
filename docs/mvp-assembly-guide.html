How the stripboard works

Hold the board with holes up, copper strips on the bottom (you solder on the copper side).

One horizontal row = one electrical connection. Every hole on that row is linked by the copper underneath. Put every wire that should be connected into any hole on the same row.

Rows do not connect to each other. Row 3 and row 4 are separate unless you add a wire between them.

Do not solder into a breadboard. On stripboard you push the wire through from the top (no copper side), bend the leg slightly, flip the board, and solder on the copper strip.

ESP32 stays off the board. Use jumper wires: female end on ESP32 pin, solder the male end into a hole on the correct row.

Stripboard matrix (one column per breakout)

Count copper rows from the top of the stripboard. Each table row is one copper trace: solder the listed pin from each column into any hole on that same row. Use rows 1–8 only; row 9+ is unused (cut shorter for the enclosure if you like). A dash means that breakout has no connection on that row.

Cable plan

The matrix above is the electrical plan. This section is the physical harness plan — what to cut or buy, and how long.

Recommended: stubs on the base + column jumpers

Do not run one 40 cm dupont all the way from a stripboard solder joint up to a remote module. Tugs in the column would stress the solder.

Instead:
1. Base stubs (~15 cm, female–male): male end soldered into the stripboard; female end sits at the base module bulkhead (zip-tied / glued so nothing pulls on the joint).
2. Column harness (~35–40 cm, male–female): male into the stub at the base; female onto the breakout pins in the upper module. Unplug here when servicing a tier.
3. ESP32 harness (~15 cm, all eight colours): same stub idea — fixed inside the base, ESP stays socketed on female ends only.

USB power is separate: one USB cable exits the base (not counted below).

Dupont gender cheat sheet: breakouts and ESP32 pins are male headers → cable needs a female dupont on that end. Stripboard wants a male pin through the hole to solder.

Rain pad only (not on stripboard)

The resistive pad uses a separate 2-wire cable (often screw terminals on the MH-RD board). Length depends on pad placement — typically 20–40 cm. Colour does not matter; polarity does not matter. Usually ships with the module; if not, any 2-core wire is fine.

One long cable per sensor? Possible for a bench test, but not recommended for the station: a single 40 cm F–M from stripboard to breakout means every snag pulls on solder. Stubs + column adds one extra plug per wire and costs more dupont, but matches how you route through the central column and makes upper modules removable without desoldering.

Build order

  1. Mark rows 1–8 on the top of the board with pencil (count from the top).
  2. Row 1 (GND): Into any holes on row 1, connect GND from ESP32 (left rail), BME280 GND, BH1750 GND, PMS5003 GND, and MH-RD GND. Spread holes so solder joints are not cramped.
  3. Row 2 (3V3): ESP32 3V3, BME280 VCC, BH1750 VCC, MH-RD VCC → row 2.
  4. Row 3 (I²C SDA, purple): ESP32 D21, BME280 SDA, BH1750 SDA → row 3.
  5. Row 4 (I²C SCL, white): ESP32 D22, BME280 SCL, BH1750 SCL → row 4.
  6. Row 5 (PMS TX→ESP, green): PMS5003 TXD + ESP32 RX2 → row 5.
  7. Row 6 (ESP TX→PMS, grey): ESP32 TX2 + PMS5003 RXD → row 6.
  8. Row 7 (5 V, red): ESP32 VIN + PMS5003 VCC → row 7.
  9. Row 8 (rain analog, blue): ESP32 D34 + MH-RD AO → row 8.
  10. Check: no solder bridges between adjacent rows (use a phone torch on the copper side).
  11. Plug USB, confirm BME/BH/rain/PM on serial or relay.
  12. Solder base stubs (15 cm F–M) per the cable plan; strain-relief stub females at the base bulkhead.
  13. Mount board in enclosure; plug column harnesses (35 cm M–F) at the bulkhead when fitting upper modules.

Flash firmware

# Any MVP env will do; the same firmware runs on every MVP board. pio run -e esp32dev_mvp1 --target upload pio device monitor -e esp32dev_mvp1

All esp32dev_mvp1 ... esp32dev_mvp5 envs build the same firmware; the per-env secrets_station_mvpN.h only seeds the first-boot defaults. Everything (WiFi, Nostr key/relay, station name, sensor toggles) is editable any time from the dashboard.

Configure from your phone

  1. Plug the station in via USB.
  2. On your phone, open WiFi settings. Join the open network named WeatherStation-XXXXXX (the suffix is the last 6 hex of the board's MAC).
  3. A "Sign in to network" sheet should pop up automatically and show the dashboard. If it doesn't, open a browser and go to http://192.168.4.1.
  4. Fill in your home WiFi, give the station a name and (optionally) a geohash location, pick which sensors you actually have wired, and hit Save. The station joins your home WiFi and starts publishing.
  5. You can reconnect to the same WeatherStation-XXXXXX network any time later to change settings. There is no app to install; nothing is hidden behind a router IP.

The setup WiFi is open (no password) and stays on even after the station is connected to your home network — that's deliberate so you can always get back in. If you forget your home WiFi password or move house, you can re-enter it the same way.
